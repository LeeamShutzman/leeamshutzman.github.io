Work Tracker

Github Repository   

What is it?

From time to time I do some work for my mother's companies, and I have to log my hours. I wanted to automate both the recording of the hours as well as the calculation my compensation, so I created a full stack project that consists of a mobile app for recording the hours, a website to view the hours worked by month and year, a Microsoft SQL Server relational database to store the data, and a Node.js server to run the website.

How does it work?

The android app first prompts the user to log in with the credentials assigned by the database admin. The passwords are stored as SHA-256 hashes in the person table in the SQL database for added security. Once the user has logged in, the user encounters the main screen that indicates that they are "Off the Clock." There is a power button in the middle of the screen that is used to start logging hours. Once this button is pressed, the background color changes and a the "Off the Clock" changes to the "On the Clock" state. The user can start and stop the clock as many times as they like throughout the day. The first time that the user turns on and off the clock on a given day, an alarm is set for 23:59. When the alarm is triggered, the phone will connect to the SQL server, upload the day's data, and reset the hours worked counter to 0.
